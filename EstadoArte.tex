\section{INTRODUÇÃO}
 Nesse capitulo será detalhado todos os passos tomados para desenvolver esse aplicativo, com isso, o cliente que tem o interesse de adquirir essa tecnologia, pode ter uma visão mais ampla desde o desenvilmento do aplicativo até a parte de aplicação, Além disso, são mostrados alguns projetos com aracteristicas semelhantes, detalhando funções melhores, que os aplicativos já tem, piores ou iguais.
 Esse aplicativo tbm serve como um banco de dados, ao qual permite que o cliente possa realizar suas compras atraves de sua própria casa. Essa forma de realizar compras é comum nos estados das grandes cidades, porém, se tiver algum comércio que utilize essa forma no Rio Grande do Norte, ainda é descohecido por nós, o que nos mostra uma maior oportunidade de edentrar nesse mundo de desenvolvimento de aplicatiivos para supermercados.
\section{MÉTODOS}
Quase todas as lojas hoje tem o auxilio de um programa com função de permitir que o supermercado tenha um gerenciamento de produtos, tornado essa tarefa mais fácil e rápida. Porém todos sabemos que há muitos mercados encontrados em diversas cidades à qual desconhecem o uso dessa tecnoliga, com isso, foi desenvolvido um aplicativo que irá permitir, facilitar e ajudar esses pequenos comercios no gerenciamento de cadastro de produtos e até mesmo em seu desenvolvimento como todo, lhe proporcionando recursos que tonará as tarefas do dia-a-dia mais facéis e por um custo razoavél. Esse aplicativo foi desenvolvido no netbeans, e utiliza o postgres para armazenar todos os cadastros e alterações realizadas no programa.\\
 NetBenas é uma plataforma que permite o programador escrever, compilar, debugar e instalar programas, passos são esses, de grande fundamentação para o desenvolvimento de qualquer que seja o programa. A maior parte do programa foi feito a partir de aplicações visuais com Swing que é uma API(Interface de Programação de Aplicativos) Java para interfaces gráficas que possibilita uma maior facilidade para o desenvolvedor de aplicativos. As APIs são recursos para tornar a programação mais rápida e reduzir a quantidade de codigos feitos pelo programador, pois elas já são prontas para serem usadas. mas mesmo utilizando o Swip, ainda sim, foi utizado POO(Programação Orientada a Objetos), principlamente para ativar os botões em geral.\\
 Esse programa contará com um sistema de gerenciamento de banco de dados que tem a função de armazenar todas as informações do produto como: Codigo, data de fabricação, data de validade, fabricante, preço, quantidade, etc. O banco utilizado foi o postgres sql.

\section{RESULTADOS ESPERADOS} 
 Com o desenvolvimento desse aplicativo, é provavel que os pequenos comercios tenha uma oportunidade maior em relação a organização, e mostrar que só porque não se trata de grandes supermercados, não possa utilizar os recursos que a tecnologia os oferece, pelo contrário, deve-se utilizar esse meio para que dessa forma o comercio possa se desenvolver aos poucos, e assim ter a oportunidade de passar de pequenos comercios para grandes supermercado.
 Também possui a responsabilidade juntamente com outro aplicativo desenvolvido para android, de permitir que as pessoas, clientes de qualquer tipo de mercado seja ele grando ou pequeno, realizar suas compras de sua própria casa, diminuindo o tempo gasto, dando mais conforto ao cliente que atravez da interação desses dois aplicativos, não possui mais toda a preocupação encarada no trânsito, como os congestionamento que tiram a paciencia do ser humsno, e reduzir a quantidade de pessoas no comercio, permitindo aos clientes não ter estresses ao caminhar pelo supermercado com um carrinho ou até mesmo na hora de ir passar os produtos que muita vezes possui uma fila enorme e um atendimento muito lento.
